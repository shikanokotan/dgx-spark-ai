# DGX Spark AI 快速上手（中文）

服务器（DGX Spark）上跑着两套**共享**的 AI 服务，大家从自己的电脑上连过去用：

| 想干嘛 | 用哪个脚本 | 是什么 |
|---|---|---|
| **画图** | `~/comfyui-connect.sh` | ComfyUI + Qwen-Image 2512（支持动漫 LoRA） |
| **聊天 / agent** | `~/nemoclaw-connect.sh` | NemoClaw 对话面板（默认 `qwen3.6:35b`） |

所有服务都跑在服务器上、只绑定本机回环地址，**不对外网开放**。你电脑上的脚本会临时开一条 SSH 隧道连过去，用完关掉就断开。模型只在服务器上加载一份，大家共用。

---

## 一、开始之前（每人一次）

1. 你需要在 Spark 上有**自己的账号**，并且配好 SSH key 能免密登录。验证一下：
   ```bash
   ssh 你的用户名@dgx.zrh.arts.moe 'echo ok'
   ```
   如果还要你输密码，先做这一步：`ssh-copy-id 你的用户名@dgx.zrh.arts.moe`
2. 把这两个脚本放到你电脑的家目录（`~/`）下，或者拿到整个 `client/` 目录都行。

> 你的账号必须已经被加入 `dgx-ai` 组（管理员 arts 操作）——NemoClaw 取登录令牌要靠它。没加的话找 arts。

---

## 二、画图：`~/comfyui-connect.sh`

直接运行：
```bash
bash ~/comfyui-connect.sh
```
**第一次**会弹出一个菜单，让你选自己是哪个账号：
```
Which DGX Spark account is yours?
   1) arts
   2) lzr
   3) yue
   4) yuki
   5) other (type a username)
Choice [1]:
```
选你自己的名字（输数字回车）。脚本接着会：确认服务器上的 ComfyUI 在跑 → 开隧道 → 自动用浏览器打开 `http://127.0.0.1:8188`。

- **这个终端窗口别关** = 保持连接；**关掉它（或按 Ctrl-C）** = 断开隧道。
- 也可以双击启动（见下面「双击打开」）。

### 画一张图
1. 左侧 **Workflows** 菜单里，打开 **`Qwen2512-<你的名字>`**（比如 `Qwen2512-lzr`）。
   选自己的那个，生成的图会存到**你自己的目录** `output/<你的名字>/`。
2. 改**正向提示词**，触发词放最前面，例如：
   `Qwen Anime, 1girl, silver hair, holding a katana, detailed hands, cherry blossoms, ...`
3. 点 **Queue**（或 Ctrl+Enter）。
4. 图会显示在界面里。第一次生成要等 ~70–80 秒（加载 20GB 模型），之后就快了。

> 说明：这是**共享单实例**，输出按用户名分了文件夹，但界面画廊里还是能看到所有人的图——属于"分门别类"，不是访问隔离。别画不想被别人看到的内容。

### 其它用法
```bash
bash ~/comfyui-connect.sh --check    # 只自检：能不能连上，然后退出
bash ~/comfyui-connect.sh --no-open  # 开隧道但不自动开浏览器（打印地址）
bash ~/comfyui-connect.sh stop       # 关掉隧道
```

## 三、聊天 / agent：`~/nemoclaw-connect.sh`

直接运行：
```bash
bash ~/nemoclaw-connect.sh
```
同样**第一次**会弹账号菜单，选自己的名字。然后脚本会：替你取一个**带令牌的面板地址** → 开隧道 → 用浏览器打开 NemoClaw 对话面板。

- 当前回答的模型：`qwen3.6:35b`（快，日常够用，默认）。想换成更强的 `gpt-oss:120b` 要管理员在服务器上切换。
- 终端窗口别关 = 保持连接；关掉 / Ctrl-C = 断开。

### 其它用法
```bash
bash ~/nemoclaw-connect.sh --check    # 自检后退出
bash ~/nemoclaw-connect.sh --no-open  # 开隧道、打印地址，不开浏览器
bash ~/nemoclaw-connect.sh stop       # 关掉隧道
```

> 聊天是**共享**的（一个助手、一套会话），没有按人隔离。

## 三点五、双击打开（不依赖终端 / Ghostty）

不想敲命令的话，直接双击 `client/` 里的：

- **`Open ComfyUI.app`** —— 画图
- **`Open NemoClaw.app`** —— 聊天

这俩是小程序，双击后由系统**直接启动**，再显式调用 **Terminal.app** 来跑脚本——**完全不经过 Ghostty 或任何其它默认终端**。所以即使你的默认终端有问题，它们照样能用，账号菜单也照常弹（Terminal 有正常的 TTY）。

> 第一次双击如果提示「未受信任的开发者 / 无法打开」，**右键点它 → 打开 → 再确认一次**即可（之后就不再提示）。

**关于那个 Ghostty 报错**：像 `ComfyUI.command` 这种 `.command` 文件，双击时是用你 Mac 的**默认终端**打开的。如果默认终端是 Ghostty 而它起不来 shell（比如报 `login -flp <用户名> …` 失败），窗口就崩了——这跟脚本无关，是 Ghostty 自己的问题。用上面的 `.app` 就能绕开它；或者把 `.command` 的默认打开方式改成 Terminal：右键 `ComfyUI.command` → 显示简介 → 「打开方式」选 **终端 (Terminal)** → 「全部更改」。

## 四、不想每次都选账号？（可选）

设一次自己的名字，之后就不再弹菜单：

**方法 A（推荐）：** 建个配置文件
```bash
cp client/dgx.conf.example ~/.config/dgx-spark.conf
# 编辑这个文件，把这一行改成你自己的：
#   : "${DGX_USER:=lzr}"
```

**方法 B：** 临时指定（只这一次）
```bash
DGX_USER=lzr bash ~/comfyui-connect.sh
DGX_USER=lzr bash ~/nemoclaw-connect.sh
```

**方法 C：** 写进 `~/.ssh/config`（脚本默认就会用对的账号）
```
Host dgx.zrh.arts.moe
    User lzr
```

## 五、常见问题

| 现象 | 原因 / 解决 |
|---|---|
| ssh 还要输密码 | SSH key 没配好：`ssh-copy-id 你的用户名@dgx.zrh.arts.moe` |
| nemoclaw 脚本报 "must be in the dgx-ai group" | 还没被加进 `dgx-ai` 组——找 arts 加，然后在服务器上**重新登录一次**生效 |
| 取令牌时提示 `sudo: a password is required` | 同上，组权限还没生效；重新登录后再试 |
| ComfyUI "did not respond" | 共享实例可能没在跑，找 arts 启动；或你在 `dgx-ai` 组里就跑：`ssh 你的用户名@dgx.zrh.arts.moe 'sudo -u arts /usr/local/bin/comfyui-ensure'` |
| 连错成了别人的账号 | 你没选对账号 / `DGX_USER` 设错了——重选，或检查 `~/.config/dgx-spark.conf` |
| Workflows 里没有 `Qwen2512-<我的名字>` | 找 arts 跑一次 `server/comfyui-add-user-workflow.sh <你的名字>`，然后刷新页面 |
